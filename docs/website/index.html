Complete Installation Guide

Follow these 4 simple steps to get TenangDB up and running

1

Install Dependencies

Test and install required tools

make test-deps
2

Get TenangDB

Clone and build the project

git clone https://github.com/abdullahainun/tenangdb.git
cd tenangdb && make build
3

Configure

Set up your database credentials

cp configs/config.yaml /etc/tenangdb/
nano /etc/tenangdb/config.yaml
4

Run First Backup

Start your automated backup system

./tenangdb backup --config /etc/tenangdb/config.yaml
🎉 That's it! Your databases are now being backed up automatically.

Production Deployment

Deploy TenangDB as a systemd service for automated, reliable database backups in production environments.

  • ⏰ Automated Scheduling - systemd timers untuk backup berkala
  • 🔐 Security Hardening - dedicated user dengan minimal privileges
  • 📊 Monitoring Ready - Prometheus metrics & Grafana dashboard
  • 🧹 Auto Cleanup - cleanup timer untuk manajemen storage
  • 📝 Centralized Logging - systemd journal integration
⏰ Configurable Backup Schedule
🛡️ Zero Root Access
📊 24/7 Monitoring
Complete Setup Guide Download Release
Production Systemd Commands
# Enable automatic backups
sudo systemctl enable tenangdb.timer
sudo systemctl start tenangdb.timer
# Enable automatic cleanup
sudo systemctl enable tenangdb-cleanup.timer
sudo systemctl start tenangdb-cleanup.timer
# Monitor backup status
sudo systemctl status tenangdb.timer
sudo journalctl -u tenangdb -f
